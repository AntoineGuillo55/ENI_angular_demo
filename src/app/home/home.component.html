On va avoir une variable ou une expression, entre double accolades, qui sera interprétée

Exemple :

10 + 2 = {{ 10 + 2 }}

Liaison par évènement

En anglais : Event Binding => Permet d'appeler les méthodes du controlleur de la vue à partir d'évènements HTML

La liaison par propriété

En anglais : Property Binding => permet de lier la valeur d'une propriété html ou css à une variable Typescript du controlleur

La liaison à double sens

En anglais : Two way binding => Permet de lier une variable Typescript à une valeur d'un champ de formulaire (input, select)

Les services

Les services sont des classes reconnues grâce au décorator "injectable". Elles peuvent donc être injectées dans d'autres classes, et même d'autres services. Ces derniers peuvent donc utiliser leurs attributs et méthodes comme si c'était les leurs.

Les services sont par défaut créés en tant que singleton.

Pour avoir un service non-singleton, il faut supprimer la propriété providedIn: 'root' du service. Ce dernier devra donc être ajouté aux classes qui l'utilisent en le spécifiant dans leur tableau de providers. Par exemple :

providers: [ImagesService]
